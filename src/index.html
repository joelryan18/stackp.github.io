[ WHAT THIS IS ]

A studio you can watch working.

Most personal sites are portfolios of finished things. stackwith.me is the opposite: every section here is a live project that keeps evolving, and the seams are left visible on purpose. When AXON's orchestration latency improves, the datasheet on its page changes. When Stackime gains a discovery rail, it ships to the same URL you visited yesterday.

The connecting thread is a single engineering philosophy: autonomy needs an audit trail, guardrails are a feature, and marketing copy should be held to product tolerance. The numbers published here are design targets we hold ourselves to — never inflated, never rounded in our favor. If a feature isn't finished, the page says so.

Everything runs on one deliberately boring, verifiable stack — a static Eleventy site, hand-written CSS on a small set of design tokens, esbuild bundles, Supabase for auth and community data, and a CDP smoke harness that must pass before anything deploys. No trackers beyond consent-gated ads, no dark patterns, no third-party fonts phoning home.

  • Transparent engineering

    Plans, specs, and trade-offs are written down before code ships — and the blog documents what actually happened, including the mistakes.

  • Honest copy, exact numbers

    Prices are exact, limits are stated, and "coming soon" means the button is disabled — not a mailing-list trap.

  • One consistent stack

    The same tokens, tooling, and quality bar power every section, so improvements in one place lift all of them.
