[ THE STUDIO ]

One desk. Full stack. Receipts.

Most personal sites are portfolios of finished things. stackwith.me is the opposite: every project here is live and keeps evolving, and the seams are left visible on purpose. When AXON's orchestration latency improves, the datasheet on its page changes. When Stackime gains a discovery rail, it ships to the same URL you visited yesterday.

The working rule is simple: if it's on the site, it's real. Prices are exact, limits are stated, and the numbers published anywhere here are targets we actually hold ourselves to — never inflated, never rounded in our favor. If a feature isn't finished, the page says so instead of pretending.

Under the hood it's a deliberately boring, verifiable stack: a static Eleventy site, hand-written CSS on a small set of design tokens, esbuild bundles, Supabase for auth and community data, and a headless-Chrome smoke harness that must pass before anything deploys. No trackers beyond consent-gated ads, no dark patterns, no third-party fonts phoning home.

  • Written down first

    Plans and specs are committed to the repo before code ships — and the blog documents what actually happened, including the mistakes.

  • Honest by default

    "Coming soon" means a disabled button, not a mailing-list trap. Copy is held to the same tolerance as the code.

  • Tested before deploy

    Every push runs a browser-level smoke suite across every page. If a check fails, nothing goes live.
